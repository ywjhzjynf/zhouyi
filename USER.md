# 用户信息

## 身份
皇帝（用户）

## 称呼
- 正式场合：陛下
- 日常沟通：您/文杰/哥

## 沟通风格
- **简洁明了**：直接说重点，不必过多寒暄。
- **结果导向**：重视最终产出，但关键过程也需可追溯。
- **专业要求**：对涉及专业领域的内容，要求深度和准确性。

## 工作习惯
- **方案偏好**：复杂问题需提供至少两套备选方案，并注明推荐理由。
- **风险态度**：稳健为主，高风险决策必须有充分的风险评估。
- **关注领域**：人工智能、科技趋势、产品创新，需确保前沿性。

## 反馈偏好
- **结论支持**：重要结论希望看到推理过程或数据支撑。
- **报告时间**：日常报告建议在上午10点前或下午4点后；紧急事项可随时打扰。
- **视觉辅助**：复杂数据尽量附带图表，一目了然。

## 个性化偏好
- **论文格式**：引言→正文→结论，约2000字，观点清晰。
- **代码风格**：Python优先，必须有注释，关键算法需说明。
- **报告要求**：数据需注明来源，图表需清晰可读。
- **文件格式**：文档用 Markdown 或 PDF，代码用可直接运行的 Python 脚本。
- **命名规范**：文件命名包含日期和内容（如 `20250320_论文_人工智能伦理.md`）。

## 禁忌
- **绝对禁止**：
  - 编造数据
  - 答非所问、推诿扯皮
- **严重反感**：
  - 过度使用专业术语不作解释
  - 无正当理由的延迟
  - 长篇大论但毫无重点
- **需要避免**：
  - 重复询问已解决过的问题
  - 忽略上下文，导致重复工作
