### 3.2 发现的严重错误

#### ❌ 错误类别 1：二进制编码错误（47 卦）

**问题描述:** 六十四卦的 `binary` 字段与上下卦组合不匹配

**错误原理:**
- 八卦二进制标准（从下往上）：
  - 乾=111, 兑=110, 离=101, 震=100
  - 巽=011, 坎=010, 艮=001, 坤=000
- 六十四卦 binary 应为：`上卦 binary + 下卦 binary`

**错误示例:**
| 卦 ID | 卦名 | 上卦 | 下卦 | 当前 binary | 正确 binary |
|-------|------|------|------|------------|------------|
| 3 | 屯 | 坎 (010) | 震 (100) | 010001 | **010100** |
| 9 | 小畜 | 巽 (011) | 乾 (111) | 110111 | **011111** |
| 10 | 履 | 乾 (111) | 兑 (110) | 111011 | **111110** |
| 15 | 谦 | 坤 (000) | 艮 (001) | 000100 | **000001** |
| 16 | 豫 | 震 (100) | 坤 (000) | 000100 | **100000** |
| ... | ... | ... | ... | ... | ... |

**完整错误列表（47 卦）:**

#### ⚠️ 发现的问题

**问题 1: HexagramCalculator.get_hexagram_number() 实现不完整**

```python
# 当前代码（第 71-86 行）
hex_map = {
    (1, 1): 1, (8, 8): 2, (6, 4): 3, (7, 4): 4,
    (6, 1): 5, (1, 6): 6, (8, 6): 7, (8, 6): 8,
    # ... 实际需要完整的 64 卦映射表
}
return hex_map.get((upper_num, lower_num), 1)
```

**问题描述:**
- 映射表不完整（只有 8 项，需要 64 项）
- 默认返回值 `1` 会导致错误卦名
- 存在重复键 `(8, 6): 7` 和 `(8, 6): 8`

**影响:** 卦名查询功能基本失效

**修复建议:**
```python
# 方案 1: 使用完整的 64 卦映射表
HEXAGRAM_MAP = {
    (1, 1): 1,   # 乾为天
    (8, 8): 2,   # 坤为地
    (6, 4): 3,   # 水雷屯
    # ... 补充完整 64 项
}

# 方案 2: 使用查表法（推荐）
# 直接从 hexagrams.json 加载数据，根据上下卦名查询
```

**问题 2: 依赖户部错误数据**

- 兵部代码逻辑正确，但依赖户部提供的 `hexagrams.json`
- 由于数据 binary 错误，导致卦象显示异常
