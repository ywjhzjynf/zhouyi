## 二、优化策略

### 2.1 JSON 压缩
- **移除空格和换行符**：使用 `separators=(',', ':')` 参数
- **效果**：减少约 15% 体积

### 2.2 字段名缩写
将冗长的英文字段名映射为缩写：

| 原字段名 | 缩写 | 节省字符 |
|---------|------|---------|
| `id` | `i` | 1 |
| `name` | `n` | 3 |
| `pinyin` | `p` | 5 |
| `full_name` | `f` | 8 |
| `symbol` | `s` | 5 |
| `binary` | `b` | 5 |
| `upper_trigram` | `u` | 12 |
| `lower_trigram` | `l` | 12 |
| `judgment` | `j` | 7 |
| `image` | `img` | 2 |
| `lines` | `ln` | 3 |
| `position` | `pos` | 5 |
| `text` | `t` | 3 |
| `type` | `tp` | 2 |
| `source` | `src` | 3 |

- **效果**：减少约 16.8% 体积
- **可逆性**：支持完全还原，不影响数据语义

### 2.3 增量加载支持
- **分块策略**：每 8 卦为一个分块（共 8 个分块）
- **索引文件**：2.4KB，包含所有卦的快速索引
- **使用场景**：
  - 首次启动：仅加载索引（2.4KB）
  - 按需加载：用户查询某卦时加载对应分块（~3.2KB/块）
  - 全量加载：一次性加载全部（25.2KB）
